1. Installation

Include Motion.js as an ES Module. No npm install required — just reference the file.

HTML
<script type="module" src="path/to/index.js"></script>

2. Declarative Animations (Zero JavaScript)

The simplest way to use Motion.js. Just add data-motion attributes to your HTML and elements animate automatically on page load. Use data-delay for sequencing.

HTML
<h1 data-motion="fade-in-up">I animate on load</h1>

<p data-motion="fade-in-down" data-delay="0.3">
  I appear 300ms later
</p>
Attribute Values Description
data-motion fade-in, fade-in-up, fade-in-down Animation type to play on load
data-delay Number (seconds) Delay before animation starts

3. JavaScript API

For full control, use the motion() function. Pass a CSS selector, DOM element, or NodeList.

JavaScript
import { motion } from './index.js';

// Animate a single element
motion('#my-box').animate(
  { opacity: 1, translateY: 0 },
  { duration: 600, ease: 'easeOutBack' }
);

4. Chaining Animations

Chain .then() to run animations sequentially. Each step waits for the previous one to complete — no manual timers needed.

JavaScript
motion('#cube')
  .animate({ scale: 1.2, rotate: 90 },  { duration: 500, ease: 'easeOutExpo' })
  .then({ scale: 0.8, rotate: 180 },    { duration: 400, ease: 'easeInOutQuad' })
  .then({ scale: 1, rotate: 360 },      { duration: 800, ease: 'easeOutBounce' });

5. Stagger Animations

When you target multiple elements (e.g. via a CSS class selector), Motion.js automatically adds a 100ms stagger delay between each element. This creates a beautiful cascading wave effect — commonly used for lists, grids, and card layouts.

JavaScript
// Each .card fades in 100ms after the previous one
motion('.card').animate(
  { opacity: 1, scale: 1 },
  { duration: 600, ease: 'easeOutBack' }
);

💡 Tip: The stagger delay stacks with the delay option. If you set delay: 200, the first element starts at 200ms, the second at 300ms, third at 400ms, etc.

6. Animatable Properties

Property Type Unit Example
opacity CSS 0–1 { opacity: 1 }
translateX Transform px { translateX: 100 }
translateY Transform px { translateY: -50 }
scale Transform unitless { scale: 1.5 }
rotate Transform deg { rotate: 180 }
skewX / skewY Transform deg { skewX: 15 }

Architecture

Motion.js is built from 5 focused modules — each under 100 lines of code.

core/engine.js

Single requestAnimationFrame loop shared by all active animations.

core/tween.js

Interpolates values from A → B over time, applying easing curves.

utils/easing.js

13 mathematical easing functions (quad, cubic, expo, back, elastic, bounce).

dom/adapter.js

Reads/writes DOM styles. Uses a WeakMap to compose multiple transforms per element.

index.js

Public API: motion() factory, chaining, and data-motion auto-initializer.
